## 🚀 Деплой на Railway (бесплатно)

### Шаг 1 — Подготовь GitHub репозиторий
1. Создай аккаунт на [github.com](https://github.com)
2. Создай новый репозиторий (например `killer-game`)
3. Загрузи все файлы проекта в репозиторий

### Шаг 2 — Настрой email для отправки кодов

**Вариант Gmail (рекомендую):**
1. Зайди в [myaccount.google.com](https://myaccount.google.com)
2. Безопасность → Двухэтапная аутентификация (включи если не включена)
3. Безопасность → Пароли приложений → создай пароль для "Почта"
4. Запиши этот пароль (16 символов) — это `MAIL_PASS`

### Шаг 3 — Деплой на Railway
1. Зайди на [railway.app](https://railway.app) → войди через GitHub
2. New Project → Deploy from GitHub repo → выбери свой репо
3. Перейди в настройки проекта → Variables → добавь переменные:

```
JWT_SECRET=придумай_длинную_строку_минимум_32_символа_например_abc123xyz789
MAIL_HOST=smtp.gmail.com
MAIL_PORT=587
MAIL_SECURE=false
MAIL_USER=твой@gmail.com
MAIL_PASS=пароль_приложения_из_google_16_символов
MAIL_FROM=Killer Game <твой@gmail.com>
APP_URL=https://твой-проект.up.railway.app
```

4. Деплой запустится автоматически
5. Зайди в Settings → Domains → Generate Domain — получишь URL игры

## 🌐 Деплой на Render (альтернатива)

1. Зайди на [render.com](https://render.com) → New → Web Service
2. Подключи GitHub репозиторий
3. Настройки:
   - **Build Command:** `npm install`
   - **Start Command:** `npm start`
4. Добавь те же переменные окружения что и для Railway
5. Нажми Deploy

> ⚠️ На бесплатном тарифе Render сервис "засыпает" после 15 минут неактивности.
> Railway этого не делает — он предпочтительнее.

## 🏠 Локальный запуск (для теста)

```bash
# 1. Установи зависимости
npm install

# 2. Создай .env файл
cp .env.example .env
# Открой .env и заполни MAIL_USER, MAIL_PASS и JWT_SECRET

# 3. Запусти сервер
npm start
# или для разработки с автоперезагрузкой:
npm run dev

# 4. Открой в браузере
# http://localhost:3000
```

## 🔑 Как работает авторизация

| Действие | Что происходит |
|----------|---------------|
| Регистрация | Создаётся аккаунт, пароль хешируется bcrypt, данные сохраняются в SQLite |
| Вход | Проверяется email+пароль, выдаётся JWT-токен (30 дней) |
| Игра | Токен хранится в localStorage, все сохранения идут на сервер |
| Восстановление | На email приходит 6-значный код (15 минут), вводишь код + новый пароль |
| Смена ника | Проверяется уникальность на сервере перед сохранением |

## 📁 Где хранится база данных

На Railway/Render база `game.db` хранится в папке проекта.
> ⚠️ При передеплое на Railway данные сохраняются. Но если Railway удалит сервис — данные пропадут.
> Для надёжности можно подключить Railway PostgreSQL (бесплатно до 1GB).
